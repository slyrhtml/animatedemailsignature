This page is a viewing aid, not the hosting solution. The images above are embedded as base64 so you can watch them animate right now — but real email clients can't reliably load base64 images like this (most strip or size-cap them). Upload the files in assets/gif/ to a public URL, then swap the thedigitxllink.com/assets/signature/ placeholder in build/signature.html for the real address before pasting the signature into Gmail, Outlook, or Apple Mail.
The card background's Outlook fallback is unverified. Outlook desktop (Windows) ignores CSS background-image entirely, so build/signature.html includes the standard VML v:rect/v:fill workaround pointing at a static glow frame instead of the animated one. This is the accepted technique for background images in Outlook, but I have no Outlook install to test it against. Please sanity-check the Outlook desktop rendering yourself, or drop the VML block and let it fall back to the solid #eaf3fc bgcolor if it causes any layout issues.
The headshot and contact text are GIFs now. That removes the risky CSS animation dependency the Claude preview exposed. The contact text is no longer selectable HTML text, but it does carry an alt description and it now follows the same baked 8-second timeline as the logo, icons, and background.
